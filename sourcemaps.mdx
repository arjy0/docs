Sourcemaps let Sonarly show you the **original source code** in error stack traces instead of minified code.

## Enable sourcemaps in your build

| Framework | Config | Sourcemaps directory |
|-----------|--------|---------------------|
| Create React App | `GENERATE_SOURCEMAP=true` in .env | `./build/static/js/` |
| Vite | `build: { sourcemap: 'hidden' }` in vite.config.ts | `./dist/assets/` |
| Next.js | `productionBrowserSourceMaps: true` in next.config.js | `./.next/static/chunks/` |
| Nuxt 3 | `sourcemap: { client: true }` in nuxt.config.ts | `./.output/public/_nuxt/` |
| Vue CLI | `productionSourceMap: true` in vue.config.js | `./dist/js/` |
| Remix | `build: { sourcemap: 'hidden' }` in vite.config.ts | `./public/build/` |
| SvelteKit | `build: { sourcemap: 'hidden' }` in vite.config.ts | `./build/_app/immutable/` |

## Upload sourcemaps

The CLI is included in `@sonarly/tracker`:

```bash
npx sonarly-upload-sourcemaps \
  --api-key YOUR_PROJECT_KEY \
  --project YOUR_PROJECT_ID \
  --server https://sonarly.dev \
  ./dist/assets  # Adjust path based on your framework
```

## Troubleshooting

### Sourcemaps not working

1. Verify `.map` files are generated in build
2. Check upload command succeeded (no 404)
3. Ensure sourcemap paths match production URLs
