Sonarly doesn't just collect errors ‚Äî it **understands** them.

## How it works

1. **Detection**: Captures errors from frontend (JS exceptions, console.error) and backend (Sentry traces)
2. **Clustering**: Groups similar errors together using AI
3. **Prioritization**: Identifies critical issues based on frequency, user impact, and severity
4. **Alerting**: Sends email alerts with full context:
   - Error message
   - File and line number
   - Function name
   - Occurrence count
   - User sessions affected

## Why AI clustering matters

Without clustering, you'd get flooded with alerts for the same issue. Sonarly's AI groups similar errors together so you only get notified once per unique problem.

- ‚úÖ 1 alert per unique issue

## Alert configuration

Configure your alert preferences in the Sonarly dashboard:

- **Severity threshold**: Only alert on critical/high issues
- **Frequency**: Immediate, hourly digest, or daily digest
- **Channels**: Email, Slack (coming soon)
- **Team notifications**: Route alerts to specific team members
