## üó∫Ô∏è Sourcemaps

Sourcemaps let Sonarly show you the **original source code** in error stack traces instead of minified code.

### Enable Sourcemaps in Your Build

| Framework | Config | Sourcemaps Directory |
|-----------|--------|---------------------|
| Create React App | `GENERATE_SOURCEMAP=true` in .env | `./build/static/js/` |
| Vite | `build: { sourcemap: true }` in vite.config.ts | `./dist/assets/` |
| Next.js | `productionBrowserSourceMaps: true` in next.config.js | `./.next/static/chunks/` |
| Nuxt 3 | `sourcemap: { client: true }` in nuxt.config.ts | `./.output/public/_nuxt/` |
| Vue CLI | `productionSourceMap: true` in vue.config.js | `./dist/js/` |
| Remix | `build: { sourcemap: true }` in vite.config.ts | `./public/build/` |
| SvelteKit | `build: { sourcemap: true }` in vite.config.ts | `./build/_app/immutable/` |

### Upload Sourcemaps

The CLI is included in `@sonarly/tracker`:

```bash
npx sonarly-upload-sourcemaps \
  --api-key YOUR_PROJECT_KEY \
  --project YOUR_PROJECT_ID \
  --server https://sonarly.dev \
  ./dist/assets  # Adjust path based on your framework
```

## üîó Backend Tracing

Sonarly captures backend traces using the **standard Sentry SDK**. No custom SDK needed!

### DSN Format

```
https://PROJECT_KEY@sonarly.dev/PROJECT_ID
```

### Node.js (Express)

```bash
npm install @sentry/node
```

```javascript
const Sentry = require("@sentry/node");

Sentry.init({
  dsn: "https://YOUR_PROJECT_KEY@sonarly.dev/PROJECT_ID",
  tracesSampleRate: 1.0,
});

const app = express();

// Link backend traces to frontend sessions
app.use((req, res, next) => {
  const sessionId = req.headers['x-sonarly-session-id'];
  if (sessionId) {
    Sentry.setTag('sonarly.session_id', sessionId);
  }
  next();
});

// Your routes here...

Sentry.setupExpressErrorHandler(app);
```

### Node.js (NestJS)

```bash
npm install @sentry/nestjs
```

```typescript
// instrument.ts
import * as Sentry from "@sentry/nestjs";

Sentry.init({
  dsn: "https://YOUR_PROJECT_KEY@sonarly.dev/PROJECT_ID",
  tracesSampleRate: 1.0,
});

// main.ts - Import instrument FIRST
import "./instrument";

// app.module.ts
import { SentryModule } from "@sentry/nestjs/setup";

@Module({
  imports: [SentryModule.forRoot()],
})
export class AppModule {}
```

### Python (Django)

```bash
pip install sentry-sdk
```

### Python (FastAPI)

```bash
pip install sentry-sdk[fastapi]
```

```python
import sentry_sdk

sentry_sdk.init(
    dsn="https://YOUR_PROJECT_KEY@sonarly.dev/PROJECT_ID",
    traces_sample_rate=1.0,
)

@app.middleware("http")
async def link_session(request, call_next):
    session_id = request.headers.get("x-sonarly-session-id")
    if session_id:
        sentry_sdk.set_tag("sonarly.session_id", session_id)
    return await call_next(request)
```

## üîî AI-Powered Alerts

Sonarly doesn't just collect errors ‚Äî it **understands** them.

### How it works

1. **Detection**: Captures errors from frontend (JS exceptions, console.error) and backend (Sentry traces)
2. **Clustering**: Groups similar errors together using AI
3. **Prioritization**: Identifies critical issues based on frequency, user impact, and severity
4. **Alerting**: Sends email alerts with full context:
   - Error message
   - File and line number
   - Function name
   - Occurrence count
   - User sessions affected

### Example Alert

```
üö® New critical issue detected in yourwebsite.com

A user is blocked after the login                    [Critical]

This AuthenticationError occurred 3 times in the last 2 minutes 
in auth.service.ts at line 145 (validateUser function).
```

## üé¨ Issue Replay

When a bug is detected, you can replay the exact session where it happened.

### What you see

- **DOM Replay**: See exactly what the user saw
- **User Actions**: Clicks, scrolls, form inputs
- **Console Logs**: All console.log, warn, error
- **Network Requests**: All fetch/XHR with request/response bodies
- **Backend Traces**: Linked server-side logs and errors

### Timeline

Errors are marked on the timeline so you can jump directly to the moment the bug occurred.

## üîß Troubleshooting

### Tracker not connecting

1. Check `projectKey` is correct
2. Check `ingestPoint` is `https://sonarly.dev/ingest`
3. Enable debug mode: `__debug__: 5`

### Sourcemaps not working

1. Verify `.map` files are generated in build
2. Check upload command succeeded (no 404)
3. Ensure sourcemap paths match production URLs

### Backend traces not appearing

1. Verify DSN format: `https://KEY@sonarly.dev/ID`
2. Set `tracesSampleRate: 1.0` (100%)
3. Restart backend after changes
4. Check CORS allows `x-sonarly-session-id` header
