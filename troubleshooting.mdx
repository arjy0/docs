## Common questions

<AccordionGroup>
  <Accordion title="How does the one-click configuration work?">
    Copy the config prompt from your Sonarly dashboard and paste it into your coding agent (Cursor, Claude, Windsurf). Your agent will automatically install the tracker, configure sourcemaps, and set up backend tracing.
  </Accordion>

  <Accordion title="What frameworks does Sonarly support?">
    Frontend: React, Vue, Svelte, Angular, and vanilla JS. Backend: Express, NestJS, Fastify, Django, FastAPI, Flask. See the [frontend](/frontend) page for details.
  </Accordion>

  <Accordion title="How do AI-powered alerts work?">
    Sonarly detects errors from frontend and backend, clusters similar issues using AI, and sends email alerts only for critical issues among Blocking, Annoying and Harmless issues.
  </Accordion>

  <Accordion title="What is LLM-ready context?">
    When a bug is detected, you can copy the full context (error, location, network requests, console logs, backend traces) and paste it directly into your coding agent to fix the bug.
  </Accordion>

  <Accordion title="Is my data secure?">
    Yes. Your data is encrypted in transit and at rest. You control what data is captured (sensitive inputs are masked by default). We never share your data with third parties.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tracker not connecting">
    Check that:
    - Your `projectKey` is correct
    - `ingestPoint` is set to `https://sonarly.dev/ingest`
    - Enable debug mode: `__debug__: 5`
    
    If issues persist, contact us at founders@sonarly.dev
  </Accordion>

  <Accordion title="Sourcemaps not working">
    Verify that:
    - `.map` files are generated in your build
    - Upload command succeeded (no 404 errors)
    - Sourcemap paths match production URLs
  </Accordion>

  <Accordion title="Backend traces not appearing">
    Check that:
    - DSN format is correct: `https://KEY@sonarly.dev/ID`
    - `tracesSampleRate` is set to `1.0`
    - Backend was restarted after configuration
    - CORS allows `x-sonarly-session-id` header (see CORS section below)
  </Accordion>

  <Accordion title="CORS errors with Sonarly tracker">
    The Sonarly tracker adds an `x-sonarly-session-id` header to your API requests. This custom header triggers CORS preflight requests.

    **Your backend must allow this header:**

    ```javascript
    // Express.js
    const cors = require('cors');

    app.use(cors({
      origin: ['https://yourdomain.com', 'https://www.yourdomain.com'],
      credentials: true,
      methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
      allowedHeaders: [
        'Content-Type',
        'Authorization',
        'x-sonarly-session-id',  // Required for Sonarly
      ],
    }));
    ```

    **Common mistakes:**
    - Missing `origin` in CORS config (causes `Access-Control-Allow-Origin` to be missing)
    - Different origins: `example.com` â‰  `www.example.com` - add both!
    - Forgetting to redeploy after CORS changes
    - Browser caching old CORS responses (try incognito or hard refresh)
  </Accordion>

  <Accordion title="Build fails with Out of Memory (OOM) on Vercel">
    Large projects with sourcemaps can exceed Vercel's 8GB memory limit.

    **Solutions (in order of effectiveness):**

    1. **Exclude source content from sourcemaps** (saves ~1GB):
    ```typescript
    // vite.config.ts
    build: {
      sourcemap: 'hidden',
      rollupOptions: {
        output: {
          sourcemapExcludeSources: true,
        }
      }
    }
    ```

    2. **Split heavy dependencies** (reduces peak memory):
    ```typescript
    manualChunks: (id) => {
      if (id.includes('node_modules')) {
        if (id.includes('firebase')) return 'firebase';
        if (id.includes('pdfjs-dist')) return 'pdf';
        // Add other heavy packages
      }
    }
    ```

    3. **Use esbuild** (default in Vite) - don't use `minify: 'terser'` which uses more memory

    4. **Increase Node memory**:
    ```bash
    NODE_OPTIONS='--max-old-space-size=8192' npm run build
    ```

    5. **Build locally and deploy pre-built** - use GitHub Actions (14GB) instead of Vercel's build

    <Note>
    Never split React core packages (`react`, `react-dom`, `scheduler`) into separate chunks - this causes runtime errors.
    </Note>
  </Accordion>

</AccordionGroup>
